 31: /**
 32:  * A <tt>ModularRealmAuthenticator</tt> delgates account lookups to a pluggable (modular) collection of
 33:  * {@link Realm}s.  This enables PAM (Pluggable Authentication Module) behavior in JSecurity.
 34:  * In addition to authorization duties, a JSecurity Realm can also be thought of a PAM 'module'.
 35:  *
 36:  * <p>Using this Authenticator allows you to &quot;plug-in&quot; your own
 37:  * <tt>Realm</tt>s as you see fit.  Common realms are those based on accessing
 38:  * LDAP, relational databases, file systems, etc.
 39:  *
 40:  * <p>If only one realm is configured (this is often the case for most applications), authentication success is naturally
 41:  * only dependent upon invoking this one Realm's
 42:  * {@link Realm#getAuthenticationInfo(org.jsecurity.authc.AuthenticationToken)} method.
 43:  *
 44:  * <p>But if two or more realms are configured, PAM behavior is implemented by iterating over the collection of realms
 45:  * and interacting with each over the course of the authentication attempt.  As this is more complicated, this
 46:  * authenticator allows customized behavior for interpreting what happens when interacting with multiple realms - for
 47:  * example, you might require all realms to be successful during the attempt, or perhaps only at least one must be
 48:  * successful, or some other interpretation.  This customized behavior can be performed via the use of a
 49:  * {@link #setModularAuthenticationStrategy(ModularAuthenticationStrategy) ModularAuthenticationStrategy}, which
 50:  * you can inject as a property of this class.
 51:  *
 52:  * <p>The strategy object provides callback methods that allow you to
 53:  * determine what constitutes a success or failure in a multi-realm (PAM) scenario.  And because this only makes sense
 54:  * in a mult-realm scenario, the strategy object is only utilized when more than one Realm is configured.
 55:  *
 56:  * <p>For greater security in a multi-realm configuration, unless overridden, the default implementation is the
 57:  * {@link AllSuccessfulModularAuthenticationStrategy AllSuccessfulModularAuthenticationStrategy}
 58:  *
 59:  * @author Jeremy Haile
 60:  * @author Les Hazlewood
 61:  * @see #setRealms
 62:  * @see AllSuccessfulModularAuthenticationStrategy
 63:  * @see AtLeastOneSuccessfulModularAuthenticationStrategy
 64:  * @since 0.1
 65:  */

239:         }

278:     /**
279:      * <p>Attempts to authenticate the given token by iterating over the internal collection of
280:      * {@link Realm}s.  For each realm, first the {@link Realm#supports(org.jsecurity.authc.AuthenticationToken)}
281:      * method will be called to determine if the realm supports the <tt>authenticationToken</tt> method argument.
282:      *
283:      * If a realm does support
284:      * the token, its {@link Realm#getAuthenticationInfo(org.jsecurity.authc.AuthenticationToken)}
285:      * method will be called.  If the realm returns a non-null account, the token will be
286:      * considered authenticated for that realm and the account data recorded.  If the realm returns <tt>null</tt>,
287:      * the next realm will be consulted.  If no realms support the token or all supporting realms return null,
288:      * an {@link AuthenticationException} will be thrown to indicate that the user could not be authenticated.
289:      *
290:      * <p>After all realms have been consulted, the information from each realm is aggregated into a single
291:      * {@link AuthenticationInfo} object and returned.
292:      *
293:      * @param authenticationToken the token containing the authentication principal and credentials for the
294:      *                            user being authenticated.
295:      * @return account information attributed to the authenticated user.
296:      * @throws AuthenticationException if the user could not be authenticated or the user is denied authentication
297:      *                                 for the given principal and credentials.
298:      */
299:     protected AuthenticationInfo doAuthenticate(AuthenticationToken authenticationToken) throws AuthenticationException {
300:         assertRealmsConfigured();
301:         Collection<Realm> realms = getRealms();
302:         if (realms.size() == 1) {
303:             return doSingleRealmAuthentication(realms.iterator().next(), authenticationToken);
304:         } else {
305:             return doMultiRealmAuthentication(realms, authenticationToken);
306:         }
307:     }
