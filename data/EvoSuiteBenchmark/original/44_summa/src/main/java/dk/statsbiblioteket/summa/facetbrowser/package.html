Provides facet browsing of search results.

Package Specification

The facet browser facilitates overview and explorative searching, by presenting a list of selected facets with tags derived from the search result.

Example

A search for eventyr (fairy tales) could yield

Authors
andersen h c 1805 (929)
grimm j l k (294)
grimm w k (286)
andersen h c (249)
walt disney firma (150)
Material types
Book (6507)
Book (paper) (6505)
Music/sound (1517)
Cassette tape (661)
Sound on CD (529)

Tags can be sorted according to popularity (e.g. the number of hits from the search result that had the given tag) or in normal alpha-numeric order.

A design criteria was complete facet browsing, which means that all hits from the search is used for creation of the overview. On a modern (by late 2006 standards) dual-processor machine, about 3 million hits can be processed per second.

The facet browser relies on a cache of the wanted facets, which takes some minutes to build for an index with a few million documents. An iterative approach to updating of the cache is on the drawing board.

Directory structure

root
The root for all index information
index-timestamp*
Lucene index and belonging indexes. The index with the highest timestamp is to be used.
Lucene index files
The files generated by Lucene.
facet-timestamp*
Facet data. The facet data with the highest timestamp is to be used.

The timestamp follows ISO standard notation: YYYMMDD-HHMMSS. The timestamps refers to creation time, not last update.
